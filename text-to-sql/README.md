# text-to-sql

Can LLMs reliably convert natural language questions into correct SQL queries?

## Question

Text-to-SQL is one of the most common enterprise AI use cases — customers want to query databases in plain English. But accuracy varies wildly depending on schema complexity, question ambiguity, and prompting strategy. How do you evaluate, debug, and systematically improve a text-to-SQL pipeline?

## Why

1. **Practical relevance** — one of the most common enterprise AI use cases. Real customers want this.
2. **Full-stack AI skills** — eval setup, prompting, error analysis, the whole loop.
3. **Python practice** — building eval pipelines, data wrangling, scripting.

## Questions

- What makes text-to-SQL hard? Where do current approaches fail?
- How do you build a good eval pipeline for text-to-SQL?
- What prompting strategies actually move the needle (schema linking, few-shot, chain-of-thought)?
- When is fine-tuning worth it vs. better prompting?
